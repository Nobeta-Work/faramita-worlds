# 编辑器增强与活跃信息展示逻辑重构方案

本方案将补全编辑器缺失字段，并重构活跃信息的展示交互。

## 1. 补全编辑器字段 (CardEditor.vue)
*   **人物卡 (Character)**：
    *   增加 `gender` (性别) 编辑项，使用 `input` 或 `select` 提供输入。
*   **设定卡 (Setting)**：
    *   增加 `category` (类别) 编辑项，支持 `background`, `race`, `level`, `class` 等预设选项。
    *   增加 `tags` (标签) 的多词条编辑功能（复用人物卡的标签列表组件）。

## 2. 活跃信息详情交互重构 (ActiveCodex.vue)
*   **全屏/窗口替换交互**：
    *   点击人物卡后，不再是在卡片下方展开详情。
    *   详情内容将**替换**掉整个“活跃信息”小窗口。
    *   在详情页顶部增加“返回列表”按钮，点击后恢复显示卡片列表。
    *   详情页将展示完整的属性、背景和标签（过滤秘密词条）。

## 3. 设定卡分类展示 (ManagerView.vue)
*   在管理界面的“设定卡”分类下，增加二级筛选（Tab 或 Dropdown），允许按 `category` (背景、种族、等级、职业) 进一步划分显示，提升管理效率。

## 4. 关键代码变更点
*   **`CardEditor.vue`**: 添加 `gender` 字段和设定卡的 `category`, `tags` 编辑 UI。
*   **`ActiveCodex.vue`**: 引入 `showDetail` 状态位，重构 `<template>` 结构，实现列表与详情的视图切换。
*   **`ManagerView.vue`**: 在设定卡视图增加二级分类过滤器。
